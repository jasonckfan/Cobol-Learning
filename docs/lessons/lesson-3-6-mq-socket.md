## 一、MQ (Message Queue) 基礎

### 1.1 什麼是 MQ？

MQ (Message Queue) 是一種訊息佇列技術，允許應用程式透過訊息進行非同步通訊。發送方將訊息放入佇列，接收方從佇列取出訊息處理。

```
┌─────────────────────────────────────────────────────────────────┐
│              MQ 訊息流程                                         │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   ┌──────────┐     PUT      ┌──────────┐     GET      ┌──────────┐│
│   │ 應用程式 A │ ───────────> │ 訊息佇列  │ ───────────> │ 應用程式 B ││
│   │ (發送方)  │              │ (Queue)  │              │ (接收方)  ││
│   └──────────┘              └──────────┘              └──────────┘│
│                                                                 │
│   特點：                                                         │
│   • 非同步通訊：發送方不需等待接收方處理                        │
│   • 解耦合：發送方和接收方不需同時在線                          │
│   • 可靠傳輸：訊息不會遺失                                      │
│   • 負載平衡：多個接收方可分擔處理                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 MQ 核心概念

| 概念 | 說明 | 範例 |
|------|------|------|
| **Queue Manager** | 佇列管理器 | 管理多個佇列的程式 |
| **Queue** | 訊息佇列 | 儲存訊息的容器 |
| **Message** | 訊息 | 傳遞的資料單位 |
| **Channel** | 通道 | 連接兩個佇列管理器的通路 |
| **PUT** | 放入 | 將訊息放入佇列 |
| **GET** | 取出 | 從佇列取出訊息 |

## 二、Socket 通訊

### 2.1 Socket 基礎

Socket 是一種網路通訊機制，允許兩個程式透過網路進行資料交換。

```
┌─────────────────────────────────────────────────────────────────┐
│              Socket 通訊模式                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   TCP/IP Socket (可靠連線)                                      │
│   ───────────────────────                                       │
│   • 建立連線 → 傳送資料 → 接收資料 → 關閉連線                  │
│   • 資料保證到達，順序一致                                      │
│   • 適合金額、交易等關鍵資料                                    │
│                                                                 │
│   UDP Socket (無連線)                                           │
│   ─────────────────                                             │
│   • 直接傳送資料包，不建立連線                                  │
│   • 不保證到達，不保證順序                                      │
│   • 適合日誌、監控等非關鍵資料                                  │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 Socket 在銀行的應用

| 應用 | 說明 | 範例 |
|------|------|------|
| **線上交易** | CICS 與後端系統通訊 | ATM 交易處理 |
| **即時查詢** | 即時餘額查詢 | 網路銀行 |
| **檔案傳輸** | 大量資料傳送 | 日終檔案交換 |
| **監控連線** | 系統狀態監控 | 心跳檢測 |

## 三、系統間資料交換模式

### 3.1 同步 vs 非同步

```
┌─────────────────────────────────────────────────────────────────┐
│              同步 vs 非同步通訊                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   同步通訊 (Synchronous)                                        │
│   ───────────────────────                                       │
│   系統 A --請求--> 系統 B                                       │
│      ↑              │                                           │
│      └────回應------┘                                           │
│                                                                 │
│   • 發送方等待回應                                              │
│   • 即時性高                                                    │
│   • 適合查詢類交易                                              │
│   • 失敗時立即知道                                              │
│                                                                 │
│   非同步通訊 (Asynchronous)                                     │
│   ─────────────────────────                                     │
│   系統 A --訊息--> 訊息佇列                                      │
│                                         ↓                       │
│   系統 A <--確認-- 訊息佇列 <--訊息-- 系統 B                    │
│                                                                 │
│   • 發送方不等待處理結果                                        │
│   • 透過佇列緩衝                                                │
│   • 適合大量交易處理                                            │
│   • 需額外機制確認處理結果                                      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 常見介面模式

| 模式 | 說明 | 適用場景 |
|------|------|----------|
| **Request/Response** | 請求回應模式 | 查詢、驗證 |
| **Fire and Forget** | 發送即忘記 | 日誌、通知 |
| **Publish/Subscribe** | 發布訂閱 | 市場資料、公告 |
| **Batch File** | 批次檔案 | 日終資料交換 |
