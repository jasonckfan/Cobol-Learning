# Lesson 5-7：上線檢查清單

## 學習目標

- 掌握上線前的準備工作
- 理解回滾計畫的重要性
- 能夠制定完整的上線檢查清單

---

## 上線準備的重要性

銀行系統上線：
- 涉及客戶資金和資料
- 影響業務運營
- 需要多方協調

**充分的準備是成功的關鍵**

---

## 上線前檢查項目

### 程式與檔案

```
□ 程式版本確認
  - 所有程式已部署到生產環境
  - 程式版本與測試環境一致
  - 相關 COPYBOOK 已更新

□ 檔案準備
  - 新檔案已建立
  - 現有檔案備份完成
  - 檔案空間已配置

□ 排程設定
  - 批次排程已安排
  - 測試執行成功
  - 相依關係確認
```

### 資料準備

```
□ 基礎資料
  - 參數檔已更新
  - 代碼檔已更新
  - 新增必要設定

□ 資料轉換
  - 歷史資料已轉換
  - 轉換結果驗證完成
  - 資料一致性確認
```

### 環境準備

```
□ 系統設定
  - 系統參數已更新
  - 權限已設定
  - 安全設定已啟用

□ 監控設定
  - 警報閾值已設定
  - 通知名單已確認
  - 監控面板已更新
```

---

## 上線執行步驟

### 上線前（D-1）

```
□ 上線會議完成
□ 所有檢查項目通過
□ 相關單位已通知
□ 備份完成
```

### 上線日（D-Day）

```
□ 業務停止（如需要）
□ 程式部署
□ 資料轉換
□ 驗證測試
□ 開放業務
```

### 上線後（D+1）

```
□ 生產驗證測試
□ 監控確認
□ 問題回報機制啟動
□ 上線報告完成
```

---

## 回滾計畫

### 為什麼需要回滾計畫？

- 上線可能失敗
- 業務無法等待
- 需要快速恢復

### 回滾計畫內容

```
1. 回滾觸發條件
   - Critical 缺陷無法即時修復
   - 業務功能嚴重異常
   - 資料正確性問題

2. 回滾步驟
   - 停止新系統服務
   - 恢復舊版程式
   - 恢復資料備份
   - 驗證系統功能
   - 通知相關單位

3. 回滾時間
   - 預估回滾時間：XX 分鐘
   - 回滾決策者：XXX
   - 回滾通知名單
```

## BA 實務技巧

### 上線會議重點

```
確認事項：
□ 上線範圍確認
□ 時程確認
□ 責任分工
□ 風險確認
□ 回滾計畫確認
□ 聯絡窗口
```

### 上線當日注意事項

```
□ 提早到達現場
□ 準備相關文件
□ 保持通訊暢通
□ 記錄執行狀況
□ 即時回報問題
```

## 練習題

### 題目 1
設計一個「新功能上線」的檢查清單，至少包含 10 個項目。

### 題目 2
說明回滾計畫應該包含哪些內容。

### 題目 3
如果上線後發現 Critical 問題，應該如何處理？

## 重點回顧

| 階段 | 檢查重點 |
|------|----------|
| 程式部署 | 版本確認、環境驗證 |
| 檔案準備 | 備份、空間配置 |
| 資料準備 | 轉換、驗證 |
| 回滾計畫 | 觸發條件、回滾步驟 |
