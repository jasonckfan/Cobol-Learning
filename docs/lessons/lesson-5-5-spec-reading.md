## 技術規格書概述

### 什麼是技術規格書？

技術規格書（Technical Specification）是將業務需求轉化為技術實作細節的文件：

```
文件轉化流程：

業務需求 (BRD)
     │
     ▼
系統需求 (SRD)
     │
     ▼
技術規格 (TSD/FSD)  ← 本課程重點
     │
     ▼
程式碼實作
```

### 為什麼 BA 需要閱讀技術規格書？

| 目的 | 說明 |
|------|------|
| 確認需求實現 | 確保技術設計符合業務需求 |
| 發現潛在問題 | 及早發現設計缺陷 |
| 溝通橋樑 | 理解開發人員的設計思路 |
| 測試設計 | 了解實作細節以設計測試案例 |
| 問題診斷 | 協助診斷生產環境問題 |

### 3. 功能規格

```markdown
功能規格範例：

## 功能編號：F-001
## 功能名稱：帳戶存款交易

### 3.1 功能概述
提供櫃員為客戶辦理存款交易的線上功能。

### 3.2 輸入規格
| 欄位名稱 | 長度 | 類型 | 必填 | 驗證規則 |
|----------|------|------|------|----------|
| 帳號 | 16 | 文字 | 是 | 數字格式，需存在於主檔 |
| 金額 | 12 | 數字 | 是 | > 0，<= 單筆限額 |
| 幣別 | 3 | 文字 | 是 | 有效幣別代碼 |
| 櫃員編號 | 8 | 文字 | 是 | 有效櫃員編號 |

### 3.3 處理邏輯
1. 驗證輸入資料格式
2. 檢查帳戶狀態（需為正常）
3. 計算新餘額
4. 更新帳戶主檔
5. 寫入交易明細
6. 回傳交易結果

### 3.4 輸出規格
| 欄位名稱 | 說明 |
|----------|------|
| 交易結果 | 成功/失敗 |
| 新餘額 | 交易後餘額 |
| 交易序號 | 本次交易序號 |
| 錯誤訊息 | 失敗時的錯誤說明 |

### 3.5 例外處理
| 錯誤代碼 | 錯誤說明 | 處理方式 |
|----------|----------|----------|
| E001 | 帳號不存在 | 顯示錯誤訊息 |
| E002 | 帳戶已凍結 | 顯示錯誤訊息 |
| E003 | 金額超過限額 | 顯示限額資訊 |
```

## 檔案規格解讀

### VSAM 檔案規格

```
檔案規格範例：

檔案名稱：ACCT.MASTER
檔案說明：存款帳戶主檔

檔案屬性：
├── 組織方式：VSAM KSDS (Key Sequenced Data Set)
├── 記錄長度：200 bytes
├── 鍵值位置：1-16
├── 鍵值長度：16 bytes
├── 編碼方式：EBCDIC
└── 壓縮方式：無

記錄格式：
各欄位定義如下：
```

```cobol
      * 帳戶主檔記錄結構
       01 ACCT-MASTER-RECORD.
          05 ACCT-NO           PIC X(16).      *> 帳號 (PK)
          05 ACCT-TYPE         PIC XX.         *> 帳戶類型
             88 ACCT-SAVINGS   VALUE 'SA'.     *>  活期存款
             88 ACCT-CURRENT   VALUE 'CA'.     *>  支票存款
             88 ACCT-FIXED     VALUE 'FD'.     *>  定期存款
          05 ACCT-NAME         PIC X(40).      *> 戶名
          05 ACCT-ID           PIC X(10).      *> 身分證字號
          05 ACCT-STATUS       PIC X.          *> 帳戶狀態
             88 ACCT-ACTIVE    VALUE 'A'.      *>  正常
             88 ACCT-FROZEN    VALUE 'F'.      *>  凍結
             88 ACCT-CLOSED    VALUE 'C'.      *>  結清
          05 ACCT-BALANCE      PIC S9(13)V99   *> 帳戶餘額
                               COMP-3.
          05 AVAIL-BALANCE     PIC S9(13)V99   *> 可用餘額
                               COMP-3.
          05 HOLD-AMOUNT       PIC S9(11)V99   *> 凍結金額
                               COMP-3.
          05 INT-RATE          PIC V9(6) COMP-3.    *> 利率
          05 ACCUM-INT         PIC S9(11)V99   *> 累計利息
                               COMP-3.
          05 OPEN-DATE         PIC 9(8).       *> 開戶日期
          05 LAST-TRX-DATE     PIC 9(8).       *> 最後交易日
          05 BRANCH-CODE       PIC X(4).       *> 開戶分行
          05 FILLER            PIC X(88).      *> 保留欄位
```

```
欄位位置計算表：

|-----------------|-------------|------|-----------|-------------|

| ACCT-TYPE       | 17-18       | 2    | XX        | 帳戶類型    |
| ACCT-NAME       | 19-58       | 40   | X(40)     | 戶名        |
| ACCT-ID         | 59-68       | 10   | X(10)     | 身分證字號  |
| ACCT-STATUS     | 69          | 1    | X         | 帳戶狀態    |
| ACCT-BALANCE    | 70-77       | 8    | PD        | 餘額(COMP-3)|
| AVAIL-BALANCE   | 78-85       | 8    | PD        | 可用餘額    |
| HOLD-AMOUNT     | 86-92       | 7    | PD        | 凍結金額    |
| INT-RATE        | 93-97       | 5    | PD        | 利率        |
| ACCUM-INT       | 98-104      | 7    | PD        | 累計利息    |
```

```cobol
      * 程式名稱：ACCTINT
      * 功能說明：日計息批次處理

       PROGRAM-ID. ACCTINT.

       INPUT-OUTPUT SECTION.
       FILE-CONTROL.
           SELECT ACCT-FILE ASSIGN TO ACCTMAS
               ORGANIZATION IS INDEXED
               ACCESS MODE IS SEQUENTIAL
               RECORD KEY IS ACCT-NO
               FILE STATUS IS WS-ACCT-STATUS.
       
       DATA DIVISION.
       FILE SECTION.
       FD ACCT-FILE.
       COPY ACCTMAST.
       
       WORKING-STORAGE SECTION.
      * 程式變數定義...
       
       PROCEDURE DIVISION.
       0000-MAIN.
           PERFORM 1000-INIT
           PERFORM 2000-PROCESS UNTIL WS-EOF = 'Y'
           PERFORM 3000-SUMMARY
           PERFORM 9000-TERM
           STOP RUN.

      *    開啟檔案
           OPEN I-O ACCT-FILE
           OPEN OUTPUT INT-FILE
           OPEN OUTPUT ERR-FILE.

      *    讀取帳戶記錄
           READ ACCT-FILE
               AT END
                   MOVE 'Y' TO WS-EOF
               NOT AT END
                   PERFORM 2100-CALC-INTEREST
           END-READ.

      *    計算利息
           IF ACCT-STATUS = 'A' AND ACCT-BALANCE > 0
               COMPUTE WS-DAILY-INT ROUNDED = 
                   ACCT-BALANCE * INT-RATE / 365
               ADD WS-DAILY-INT TO ACCUM-INT
               PERFORM 2200-WRITE-INT-RECORD
           END-IF.
       
       2200-WRITE-INT-RECORD.
      *    寫入利息記錄
           MOVE ACCT-NO TO INT-ACCT-NO
           MOVE WS-DAILY-INT TO INT-AMT
           MOVE WS-TODAY TO INT-DATE
```

## BA 實務技巧

### 規格書閱讀步驟

```
Step 1: 先看概述（5 分鐘）
├── 目的與範圍
├── 系統架構
└── 功能清單

Step 2: 看功能規格（30 分鐘）
├── 功能描述
├── 輸入/輸出規格
├── 處理邏輯
└── 例外處理

Step 3: 看資料規格（20 分鐘）
├── 檔案結構
├── 欄位定義
└── 資料關聯

Step 4: 看介面規格（15 分鐘）
├── 系統介面
├── 訊息格式
└── 錯誤處理

Step 5: 對照需求規格（20 分鐘）
├── 確認需求實現
├── 找出差異
└── 提出問題
```

### 規格書審核重點

```
完整性審核：
□ 是否涵蓋所有需求？
□ 是否有遺漏的情境？
□ 例外處理是否完整？

一致性審核：
□ 前後描述是否一致？
□ 與需求規格是否一致？
□ 與現有系統是否相容？

可行性審核：
□ 技術方案是否可行？
□ 效能要求是否合理？
□ 時程是否可達成？

安全性審核：
□ 是否符合安全規範？
□ 敏感資料是否保護？
□ 權限控制是否完整？
```

## 重點回顧

| 規格類型 | 關注重點 | 常見問題 |
|----------|----------|----------|
| 系統架構 | 元件關係、資料流向 | 介面遺漏 |
| 功能規格 | 輸入輸出、處理邏輯 | 例外處理不完整 |
| 檔案規格 | 組織方式、欄位定義 | 長度計算錯誤 |
| 程式規格 | 輸入輸出檔案、處理邏輯 | 效能問題 |
| 介面規格 | 訊息格式、錯誤代碼 | 版本不相容 |
