# Lesson 5-5：技術規格書閱讀

## 學習目標

- 理解技術規格書的結構
- 能夠解讀關鍵技術資訊
- 掌握確認規格的方法

---

## 技術規格書結構

### 常見結構

```
1. 文件控制
   - 版本歷史
   - 審核記錄

2. 概述
   - 目的
   - 範圍
   - 定義與縮寫

3. 系統架構
   - 整體架構圖
   - 系統元件說明

4. 功能規格
   - 功能清單
   - 詳細功能說明
   - 畫面規格
   - 處理邏輯

5. 資料規格
   - 資料結構
   - 檔案規格
   - 資料庫規格

6. 介面規格
   - 系統介面
   - 訊息格式

7. 非功能需求
   - 效能需求
   - 安全需求
   - 可用性需求
```

## 重要欄位解讀

### 檔案規格

```
檔案名稱：ACCT.MASTER
組織方式：VSAM KSDS
記錄長度：200 bytes
鍵值位置：1-16
鍵值長度：16

欄位定義：
| 名稱        | 位置 | 長度 | 格式   | 說明     |
|-------------|------|------|--------|----------|
| ACCT-NO     | 1    | 16   | X      | 帳號(主鍵)|
| ACCT-NAME   | 17   | 40   | X      | 戶名     |
| ACCT-BALANCE| 57   | 8    | PD     | 餘額     |
```

### 程式規格

```
程式名稱：ACCTINT
程式類型：批次 (Batch)
執行頻率：每日

輸入檔案：
- ACCT.MASTER (帳戶主檔)

輸出檔案：
- INT.DAILY (利息明細)

處理邏輯：
1. 讀取帳戶主檔
2. 計算利息 = 餘額 × 利率 / 365
3. 寫入利息明細
4. 更新累計利息
```

### 標準符號

```
┌─────┐
│開始/│  開始/結束
└─────┘

┌─────────┐
│  處理   │  處理步步驟
└─────────┘

◇  決策點

─────▶  流向箭頭
```

## 關鍵問題確認

### 檔案相關

```
□ 檔案組織方式是什麼？
□ 記錄長度是多少？
□ 鍵值是什麼？
□ 有哪些欄位？各佔多少位置？
```

### 程式相關

```
□ 程式類型是什麼？（批次/線上）
□ 輸入檔案有哪些？
□ 輸出檔案有哪些？
□ 主要處理邏輯是什麼？
□ 有哪些錯誤處理？
```

### 介面相關

```
□ 通訊方式是什麼？（MQ/Socket/File）
□ 訊息格式是什麼？
□ 回應時間要求是多少？
```

## BA 實務技巧

### 規格書閱讀步驟

```
1. 先看概述，了解整體目的
2. 看系統架構，了解各元件關係
3. 看功能規格，了解業務邏輯
4. 看資料規格，了解資料結構
5. 看介面規格，了解系統互動
```

### 規格書審核重點

```
□ 是否與需求一致？
□ 是否有遺漏？
□ 是否有矛盾？
□ 是否可測試？
□ 是否有風險？
```
