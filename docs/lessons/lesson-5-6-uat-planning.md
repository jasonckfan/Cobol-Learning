# Lesson 5-6：UAT 測試規劃

## 學習目標

- 理解 UAT 的目的與流程
- 能夠規劃 UAT 測試
- 掌握測試數據準備方法

---

## 什麼是 UAT？

UAT（User Acceptance Testing，使用者驗收測試）是：

- 由業務單位執行的測試
- 驗證系統是否符合業務需求
- 上線前的最後一道關卡

---

## UAT 流程

### 執行步驟

```
1. 測試規劃
   - 確定測試範圍
   - 設計測試案例
   - 準備測試數據

2. 測試執行
   - 執行測試案例
   - 記錄測試結果
   - 回報缺陷

3. 缺陷修復
   - 開發單位修正
   - 重新測試驗證

4. 驗收簽核
   - 業務單位確認
   - 正式簽核
```

---

## 測試案例設計

### UAT 測試案例特點

- 以業務場景為導向
- 使用真實業務數據
- 涵蓋端到端流程

### 測試案例範例

```markdown
## 測試案例：TC-UAT-001
## 測試場景：客戶臨櫃存款

### 前置條件：
- 櫃員已登入系統
- 客戶帳號存在且狀態正常
- 客戶持有現金 10,000 元

### 測試步驟：
1. 選擇「存款」交易
2. 輸入客戶帳號：1234567890123456
3. 輸入存款金額：10,000
4. 確認交易
5. 列印交易憑條

### 預期結果：
- 交易成功
- 帳戶餘額增加 10,000
- 產生交易明細
- 列印憑條內容正確
```

## 測試數據準備

### 測試數據類型

| 類型 | 說明 | 範例 |
|------|------|------|
| 正常數據 | 正常業務情境 | 一般帳戶、正常交易 |
| 邊界數據 | 邊界條件測試 | 限額邊界值 |
| 異常數據 | 異常情境測試 | 凍結帳戶、餘額不足 |
| 特殊數據 | 特殊情境測試 | 週末、月底、年底 |

### 測試帳戶準備

```
| 帳號              | 狀態   | 餘額      | 用途           |
|-------------------|--------|-----------|----------------|
| 1234567890123456  | 正常   | 100,000   | 正常交易測試   |
| 1234567890123457  | 正常   | 100       | 餘額不足測試   |
| 1234567890123458  | 凍結   | 50,000    | 凍結帳戶測試   |
| 1234567890123459  | 結清   | 0         | 結清帳戶測試   |
```

## 測試環境安排

### 環境需求

| 項目 | 說明 |
|------|------|
| 測試環境 | UAT 專用環境，與生產環境隔離 |
| 測試數據 | 模擬真實數據 |
| 測試工具 | 測試管理工具、缺陷追蹤工具 |
| 測試帳號 | 測試人員專用帳號 |

### 測試排程

```
| 階段         | 日期       | 工作項目           |
|--------------|------------|---------------------|
| 準備階段     | 4/1 - 4/3  | 測試案例、數據準備  |
| 執行階段     | 4/4 - 4/8  | 執行測試案例        |
| 缺陷修復     | 4/9 - 4/12 | 缺陷修正與驗證      |
| 回歸測試     | 4/13 - 4/15| 回歸測試           |
| 驗收簽核     | 4/16       | 業務簽核           |
```

## 缺陷追蹤

### 缺陷分級

| 等級 | 說明 | 處理時效 |
|------|------|----------|
| Critical | 系統無法使用 | 立即處理 |
| Major | 主要功能無法使用 | 24 小時內 |
| Minor | 次要功能異常 | 48 小時內 |
| Cosmetic | 畫面/文案問題 | 上線前修復 |

### 缺陷報告

```markdown
## 缺陷編號：DEF-001
## 嚴重程度：Major

### 問題描述：
存款交易執行後，餘額未即時更新

### 重現步驟：
1. 登入系統
2. 執行存款 10,000 元
3. 查詢帳戶餘額

### 預期結果：
餘額應增加 10,000

### 實際結果：
餘額未變更

### 測試環境：
UAT 環境 / 測試帳號 1234567890123456

### 附件：
- 畫面截圖
- 交易明細
```

## BA 實務技巧

### UAT 檢查清單

```
測試前：
□ 測試案例已完成
□ 測試數據已準備
□ 測試環境已就緒
□ 測試人員已安排

測試中：
□ 每日測試進度追蹤
□ 缺陷及時回報
□ 缺陷修復驗證

測試後：
□ 測試報告完成
□ 缺陷清單確認
□ 業務簽核完成
```

### 測試完成準則

```
□ 所有測試案例已執行
□ Critical/Major 缺陷已修復
□ Minor 缺陷已有處理計畫
□ 業務單位已確認驗收
```
