# Lesson 5-2：影響分析方法

## 學習目標

- 掌握影響分析的方法論
- 能夠識別需求變更的影響範圍
- 理解銀行系統的依賴關係

---

## 什麼是影響分析？

影響分析是評估變更對現有系統影響的過程：

- 識別受影響的程式和檔案
- 評估變更的風險和複雜度
- 估算開發和測試工作量

---

## 影響分析步驟

### 1. 變更識別

```
變更類型：
- 新增功能
- 修改功能
- 刪除功能
- 資料結構變更
```

### 2. 影響範圍識別

```
識別維度：
- 程式影響
- 檔案影響
- 介面影響
- 批次影響
- 報表影響
```

### 3. 風險評估

```
風險等級：
- 高：核心功能、大量程式受影響
- 中：重要功能、部分程式受影響
- 低：次要功能、少數程式受影響
```

## 影響分析實例

### 案例：帳戶主檔新增「手機號碼」欄位

#### Step 1: 檔案結構變更

```cobol
      * 變更前
       01 ACCT-RECORD.
          05 ACCT-NO           PIC X(16).
          05 ACCT-NAME         PIC X(40).
          05 ACCT-BALANCE      PIC S9(11)V99 COMP-3.
          05 FILLER            PIC X(20).

      * 變更後
       01 ACCT-RECORD.
          05 ACCT-NO           PIC X(16).
          05 ACCT-NAME         PIC X(40).
          05 ACCT-MOBILE       PIC X(15).      *> 新增
          05 ACCT-BALANCE      PIC S9(11)V99 COMP-3.
          05 FILLER            PIC X(5).       *> 縮減
```

#### Step 2: 受影響程式識別

| 程式類型 | 程式名稱 | 影響說明 |
|----------|----------|----------|
| 線上交易 | ACCTOPEN | 開戶時需輸入手機 |
| 線上交易 | ACCTMAINT | 維護手機號碼 |
| 線上交易 | ACCTQUERY | 顯示手機號碼 |
| 批次 | DAILYINT | 讀取但不處理 |
| 批次 | ACCTRPT | 報表顯示手機 |
| COPYBOOK | ACCTREC | 結構定義變更 |

#### Step 3: 工作量估算

| 工作項目 | 工時 | 說明 |
|----------|------|------|
| COPYBOOK 修改 | 0.5 天 | 資料結構變更 |
| 線上程式修改 | 3 天 | 3 支程式 |
| 批次程式修改 | 1 天 | 報表調整 |
| 測試 | 2 天 | 功能測試 + 回歸測試 |
| **總計** | **6.5 天** | |

## BA 實務技巧

### 影響分析檢查清單

```
□ COPYBOOK 變更
  - 檔案長度是否改變？
  - 是否影響現有欄位位置？
  - 使用此 COPYBOOK 的所有程式？

□ 線上交易變更
  - 畫面是否需要修改？
  - 輸入驗證邏輯？
  - 輸出顯示邏輯？

□ 批次程式變更
  - 排程是否需要調整？
  - 相依程式是否受影響？
  - 報表格式是否需要修改？

□ 介面變更
  - 對外介面是否受影響？
  - 是否需要通知外部單位？
  - 版本相容性？
```

### 風險評估矩陣

| 影響範圍 | 變更複雜度 | 風險等級 |
|----------|------------|----------|
| 單一程式 | 簡單 | 低 |
| 單一程式 | 複雜 | 中 |
| 多個程式 | 簡單 | 中 |
| 多個程式 | 複雜 | 高 |
| 核心功能 | 任何 | 高 |

## 練習題

### 題目 1
如果要在交易明細檔新增「備註」欄位，請列出可能受影響的程式類型。

### 題目 2
設計一個影響分析表，評估「新增客戶等級欄位」的變更。

### 題目 3
說明為什麼 COPYBOOK 變更需要重新編譯所有使用它的程式。

## 重點回顧

| 步驟 | 說明 |
|------|------|
| 變更識別 | 確認變更類型和內容 |
| 影響範圍識別 | 找出所有受影響的程式和檔案 |
| 風險評估 | 評估變更的風險等級 |
| 工作量估算 | 估算開發和測試工時 |
