# Lesson 5-2：影響分析方法

## 學習目標

- 掌握影響分析的方法論與工具
- 能夠識別需求變更的影響範圍
- 理解銀行系統的依賴關係
- 能夠進行風險評估與工作量估算

---

## 什麼是影響分析？

### 定義

影響分析（Impact Analysis）是評估變更對現有系統影響的過程：

```
影響分析的目的：
├── 識別受影響的程式和檔案
├── 評估變更的風險和複雜度
├── 估算開發和測試工作量
├── 制定變更實施計計畫
└── 降低變更帶來的風險
```

### 為什麼重要？

| 情境 | 不做影響分析的後果 |
|------|---------------------|
| 檔案結構變更 | 遺漏受影響程式，導致批次失敗 |
| 業務規則變更 | 測試覆蓋不全，上線後發現問題 |
| 介面變更 | 影響外部系統，造成客戶投訴 |
| 效能調整 | 影響其他交易，系統變慢 |

## 影響分析步驟

### Step 1: 變更識別

```
變更類型：
├── 資料結構變更
│   ├── 新增欄位
│   ├── 修改欄位長度/格式
│   ├── 刪除欄位
│   └── 檔案結構重組
│
├── 功能變更
│   ├── 新增功能
│   ├── 修改功能
│   ├── 刪除功能
│   └── 業務規則變更
│
├── 介面變更
│   ├── API 變更
│   ├── 檔案格式變更
│   └── 訊息格式變更
│
└── 非功能變更
    ├── 效能優化
    ├── 安全性增強
    └── 合規性調整
```

### Step 2: 影響範圍識別

```
識別維度：
├── 程式影響
│   ├── 直接使用（讀取/更新）
│   ├── 間接影響（透過 COPYBOOK）
│   └── 相依程式（呼叫鏈）
│
├── 檔案影響
│   ├── 結構變更
│   ├── 資料遷移
│   └── 備份/回復
│
├── 介面影響
│   ├── 對外介面
│   ├── 內部介面
│   └── 版本相容
│
├── 批次影響
│   ├── 排程依賴
│   ├── 處理順序
│   └── 效能影響
│
└── 報表影響
    ├── 報表格式
    ├── 資料來源
    └── 輸出變更
```

### Step 3: 風險評估

```
風險評估矩陣：

影響範圍 ──────────────────────────────
  │
高 │  中風險           高風險          高風險
  │  (多程式/簡單)     (多程式/複雜)   (核心功能)
  │
中 │  低風險           中風險          高風險
  │  (少程式/簡單)     (少程式/複雜)   (重要功能)
  │
低 │  低風險           低風險          中風險
  │  (單程式/簡單)     (單程式/複雜)   (次要功能)
  │
  └──────────────────────────────────────
        低               中              高
              變更複雜度
```

### Step 4: 工作量估算

```
估算項目：
├── 開發工作量
│   ├── 程式修改
│   ├── COPYBOOK 修改
│   ├── JCL 修改
│   └── 畫面修改
│
├── 測試工作量
│   ├── 單元測試
│   ├── 整合測試
│   ├── 系統測試
│   └── 回歸測試
│
├── 移轉工作量
│   ├── 資料遷移
│   ├── 環境建置
│   └── 上線準備
│
└── 文件工作量
    ├── 規格書更新
    ├── 操作手冊更新
    └── 訓練材料
```

## 影響分析實例

### 案例 1：帳戶主檔新增「手機號碼」欄位

#### Step 1: 檔案結構變更

```cobol
      * 變更前
       01 ACCT-RECORD.
          05 ACCT-NO           PIC X(16).
          05 ACCT-NAME         PIC X(40).
          05 ACCT-BALANCE      PIC S9(13)V99 COMP-3.
          05 ACCT-STATUS       PIC X.
          05 FILLER            PIC X(20).

      * 變更後
       01 ACCT-RECORD.
          05 ACCT-NO           PIC X(16).
          05 ACCT-NAME         PIC X(40).
          05 ACCT-MOBILE       PIC X(15).      *> 新增
          05 ACCT-BALANCE      PIC S9(13)V99 COMP-3.
          05 ACCT-STATUS       PIC X.
          05 FILLER            PIC X(5).       *> 縮減
```

#### Step 2: 受影響程式識別

```
影響識別工具查詢：

使用 COPYBOOK: ACCTREC
├── 程式清單:
│   ├── ACCTOPEN.cbl   - 開戶交易
│   ├── ACCTMAINT.cbl  - 帳戶維護
│   ├── ACCTQUERY.cbl  - 帳戶查詢
│   ├── ACCTCLOSE.cbl  - 帳戶結清
│   ├── DAILYINT.cbl   - 日計息批次
│   ├── MONTHINT.cbl   - 月計息批次
│   ├── ACCTRPT.cbl    - 帳戶日報
│   ├── ACCTLIST.cbl   - 帳戶清單
│   └── ACCTRECON.cbl  - 帳戶對帳
│
└── 總計: 9 支程式受影響
```

#### Step 3: 影響分析表

| 程式名稱 | 類型 | 影響說明 | 修改內容 | 工時 |
|----------|------|----------|----------|------|
| ACCTCOPY | COPYBOOK | 結構定義 | 新增欄位 | 0.5天 |
| ACCTOPEN | 線上交易 | 開戶輸入 | 輸入手機、驗證格式 | 1.5天 |
| ACCTMAINT | 線上交易 | 帳戶維護 | 修改手機功能 | 1天 |
| ACCTQUERY | 線上交易 | 帳戶查詢 | 顯示手機 | 0.5天 |
| ACCTCLOSE | 線上交易 | 帳戶結清 | 結清報表顯示手機 | 0.5天 |
| DAILYINT | 批次 | 日計息 | 讀取但不處理 | 0.25天 |
| MONTHINT | 批次 | 月計息 | 讀取但不處理 | 0.25天 |
| ACCTRPT | 報表 | 日報表 | 新增欄位 | 0.5天 |
| ACCTLIST | 報表 | 帳戶清單 | 新增欄位 | 0.5天 |
| ACCTRECON | 批次 | 對帳 | 不影響 | 0天 |
| **測試** | 測試 | 回歸測試 | 全部受影響程式 | 3天 |
| **總計** | | | | **8.5天** |

#### Step 4: 資料遷移規劃

```
資料遷移步驟：

Step 1: 備份現有資料
        - 備份 ACCT-MASTER 檔案
        - 確認備份完整性

Step 2: 建立新檔案結構
        - 定義新的 VSAM Cluster
        - 設定新的欄位配置

Step 3: 轉換資料
        - 讀取舊檔案
        - 新增欄位填入預設值（空白）
        - 寫入新檔案

Step 4: 驗證資料
        - 筆數核對
        - 金額核對
        - 抽樣檢查

Step 5: 切換
        - 停止相關交易
        - 執行資料遷移
        - 驗證結果
        - 開啟交易
```

### 案例 2：交易金額限額調整

#### 業務需求

> 「配合詐騙防制，調整網銀轉帳限額：單筆從 50,000 降為 30,000」

#### 影響分析

| 變更項目 | 受影響程式 | 影響說明 | 工時 |
|----------|------------|----------|------|
| 參數檔調整 | LIMIT-PARAM | 參數設定 | 0.5天 |
| 網銀交易 | WEBTRANSFER | 限額檢核 | 0.5天 |
| 手機 APP | APPTRANSFER | 限額檢核 | 0.5天 |
| 後台管理 | LIMITMAINT | 參數維護畫面 | 0.5天 |
| 警示通知 | LIMITALERT | 超額警示 | 0.5天 |
| 測試 | 測試 | 功能測試 + 回歸測試 | 2天 |
| **總計** | | | **4.5天** |

#### 注意事項

```
變更注意事項：
├── 對外溝通
│   ├── 公告客戶（提前 30 天）
│   ├── 更新網頁說明
│   └── 客服教育訓練
│
├── 系統調整
│   ├── 參數檔版本管理
│   ├── 新舊限額過渡期處理
│   └── 緊急回復機制
│
└── 測試重點
    ├── 新限額生效測試
    ├── 邊界值測試（29,999, 30,000, 30,001）
    ├── 歷史交易查詢（舊限額記錄）
    └── 效能測試（確認無影響）
```

#### 業務需求

| 線上交易修改 | 3天 | 轉帳交易、畫面調整 |
| 批次程式修改 | 1天 | 日終批次 |
| 報表修改 | 1天 | 交易明細報表 |
| AML 系統介接 | 2天 | 資料提供 |
| 測試 | 3天 | 功能 + 回歸測試 |
| **總計** | **11天** | |

## BA 實務技巧

### 影響分析檢核清單

```
COPYBOOK 變更檢核：
□ 確認所有使用此 COPYBOOK 的程式
□ 檔案長度是否改變？影響 VSAM 重建？
□ 是否影響現有欄位位置？
□ 是否需要資料遷移？
□ 測試環境需要哪些調整？

線上交易變更檢核：
□ CICS 資源定義是否需要調整？
□ 畫面是否需要修改？
□ 輸入驗證邏輯？
□ 輸出顯示邏輯？
│ 效能影響評估？
□ 交易回復機制？

批次程式變更檢核：
□ JCL 是否需要修改？
□ 排程是否需要調整？
□ 相依程式是否受影響？
□ 報表格式是否需要修改？
□ 效能影響評估？
□ 錯誤處理機制？

介面變更檢核：
□ 對外介面是否受影響？
□ 是否需要通知外部單位？
□ 版本相容性？
│ 向後相容性？
□ 測試環境模擬？
```

### 風險緩解措施

| 風險 | 緩解措施 |
|------|----------|
| 遺漏受影響程式 | 使用工具自動掃描，人工覆核 |
| 資料遷移失敗 | 準備回復計畫，分批執行 |
| 效能下降 | 效能基準測試，監控機制 |
| 測試覆蓋不足 | 制定完整測試計畫 |
| 上線失敗 | 準備回退方案 |

### 溝通重點

```
與開發團隊溝通：
├── 明確說明變更範更範圍
├── 提供完整的影響分析表
├── 討論技術實作方案
└── 確認工作量估算

與業務單位溝通：
├── 說明變更的複雜度
├── 解釋需要時間的原因
├── 討論風險與緩解措施
└── 確認上線時程

與管理層溝通：
├── 彙總工作量與成本
├── 說明風險等級
├── 提出資源需求
└── 建議實施時程
```
