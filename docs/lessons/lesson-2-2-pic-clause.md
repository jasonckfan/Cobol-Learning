# Lesson 2-2：變數宣告與 PIC 子句

## 學習目標

- 理解 COBOL 變數宣告語法
- 掌握 PIC 子句的各種格式
- 能夠解讀銀行程式中的數值與文字欄位

---

## 為什麼 PIC 子句很重要？

在銀行程式中，每個欄位都有嚴格定義：
- 帳號幾碼？用什麼格式？
- 金額最大多少？有沒有小數？
- 日期怎麼表示？

PIC 子句就是用來定義這些規格的，**BA 看懂 PIC 就能理解資料結構**。

---

## 基本語法

```cobol
       01 變數名稱 PIC 格式.
```

或簡寫為：

```cobol
       01 變數名稱 PIC 格式.
```

---

## 文字欄位（Alphanumeric）

### 基本格式

| PIC 格式 | 說明 | 範例值 |
|----------|------|--------|
| `X(n)` | n 個字元（可為任何字元） | `X(10)` = 10 個字元 |
| `A(n)` | n 個字母（A-Z, 空白） | `A(20)` = 20 個字母 |
| `X` | 1 個字元 | `X` = 單一字元 |

### 範例

```cobol
       01 ACCT-NO            PIC X(10).    *> 帳號，10碼
       01 ACCT-NAME          PIC X(30).    *> 戶名，30字元
       01 ACCT-TYPE          PIC XX.       *> 帳戶類型，2碼
       01 STATUS-FLAG        PIC X.        *> 狀態旗標，1碼
```

> 🏦 **銀行實例**：
> - 帳號通常 10-16 碼
> - 戶名通常 30-50 字元
> - 幣別代碼 3 碼（USD, HKD, CNY）

---

## 數值欄位（Numeric）

### 基本格式

| PIC 格式 | 說明 | 範圍 |
|----------|------|------|
| `9(n)` | n 位整數 | 0 到 10^n - 1 |
| `9(n)V9(m)` | n 位整數 + m 位小數 | V 是小數點位置（隱含） |
| `S9(n)` | 帶正負號整數 | -10^n 到 10^n - 1 |
| `S9(n)V9(m)` | 帶正負號小數 | 含小數的正負數 |

### 重要觀念

**V 是「虛擬小數點」**：
- 實際資料中不儲存小數點
- 程式運算時自動處理小數位置
- 銀行金額欄位必備！

### 範例

```cobol
       01 ACCT-BALANCE       PIC S9(9)V99.    *> 餘額：最多 9 位整數，2 位小數，可正負
       01 TRANS-AMT          PIC S9(7)V99.    *> 交易金額
       01 INT-RATE           PIC V9999.       *> 利率：4 位小數（如 0.0250）
       01 ACCOUNT-COUNT      PIC 9(5).        *> 帳戶數量（無負數）
       01 CUSTOMER-ID        PIC 9(8).        *> 客戶 ID（純數字）
```

### 銀行常見數值欄位

| 欄位類型 | PIC 格式 | 說明 |
|----------|----------|------|
| 帳戶餘額 | `S9(11)V99` | 最大 9,999,999,999.99 |
| 利率 | `V9(6)` | 如 0.027500 |
| 交易金額 | `S9(9)V99` | 最大 999,999,999.99 |
| 序號 | `9(8)` | 如流水號 |
| 匯率 | `S9(5)V9(6)` | 如 7.850000 |

## 編輯符號（Editing Characters）

當需要輸出報表或顯示時，使用編輯符號來格式化數值。

### 常用編輯符號

| 符號 | 說明 | 範例 |
|------|------|------|
| `Z` | 抑制前導零 | `ZZZ9` → `   5` |
| `,` | 千分位逗號 | `9,999` → `1,234` |
| `.` | 小數點（實際顯示） | `999.99` → `123.45` |
| `$` | 貨幣符號 | `$9,999.99` → `$1,234.56` |
| `-` | 負號（僅負數顯示） | `-999` → `-123` 或 ` 123` |
| `+` | 正負號都顯示 | `+999` → `+123` 或 `-123` |
| `CR` | 貸方（負數顯示 CR） | `999CR` → `123  ` 或 `123CR` |
| `DB` | 借方（負數顯示 DB） | `999DB` |

### 範例

```cobol
       01 WS-BALANCE-EDIT    PIC $$$,$$$,$$9.99.    *> 格式化餘額
       01 WS-RATE-EDIT       PIC 9.9999%.            *> 利率顯示
       01 WS-AMT-EDIT        PIC Z(8)9.99-.          *> 金額帶負號
```

### 轉換範例

```
原始值: 0001234567.89

PIC 9(10)V99        → 000123456789  (無格式)
PIC Z(9)9.99        →    1234567.89  (去零加小數點)
PIC $Z,ZZZ,ZZ9.99   → $1,234,567.89  (貨幣格式)
```

## 重複次數的簡寫

### 格式

```cobol
       PIC X(10)          *> 等於 PIC XXXXXXXXXX
       PIC 9(5)           *> 等於 PIC 99999
       PIC S9(7)V99       *> 等於 PIC S9999999V99
```

## 實際銀行程式範例

### 帳戶主檔記錄

```cobol
       01 ACCT-MASTER-REC.
          05 ACCT-NO           PIC X(16).           *> 帳號
          05 ACCT-TYPE         PIC XX.              *> 帳戶類型（SA/CA/FD）
          05 ACCT-NAME         PIC X(40).           *> 戶名
          05 ACCT-STATUS       PIC X.               *> 狀態（A/D/S）
          05 OPEN-DATE         PIC 9(8).            *> 開戶日期（YYYYMMDD）
          05 CURR-CODE         PIC XXX.             *> 幣別
          05 ACCT-BALANCE      PIC S9(11)V99.       *> 帳戶餘額
          05 HOLD-AMT          PIC S9(9)V99.        *> 凍結金額
          05 AVAIL-BAL         PIC S9(11)V99.       *> 可用餘額
          05 INT-RATE          PIC V9(6).           *> 利率
          05 LAST-TRX-DATE     PIC 9(8).            *> 最後交易日
          05 FILLER            PIC X(20).           *> 保留欄位
```

### 交易明細記錄

```cobol
       01 TRANS-RECORD.
          05 TRANS-DATE        PIC 9(8).            *> 交易日期
          05 TRANS-TIME        PIC 9(6).            *> 交易時間（HHMMSS）
          05 TRANS-TYPE        PIC XX.              *> 交易類型
          05 ACCT-NO           PIC X(16).           *> 帳號
          05 TRANS-AMT         PIC S9(9)V99.        *> 交易金額
          05 BALANCE-AFTER     PIC S9(11)V99.       *> 交易後餘額
          05 REF-NO            PIC X(20).           *> 參考編號
          05 REMARK            PIC X(50).           *> 備註
```

## BA 實務應用

### 如何從 PIC 判斷欄位用途？

| PIC 格式 | 可能用途 |
|----------|----------|
| `X(10)` ~ `X(20)` | 帳號、客戶編號 |
| `X(30)` ~ `X(50)` | 姓名、地址 |
| `X(2)` ~ `X(4)` | 類型代碼、幣別 |
| `9(8)` | 日期（YYYYMMDD） |
| `9(6)` | 時間（HHMMSS） |
| `S9(n)V99` | 金額（可正負） |
| `V9(n)` | 利率、比率 |

### 需求分析時的關鍵問題

**情境：新增「客戶手機號碼」欄位**

1. 手機號碼長度？（香港 8 碼，大陸 11 碼）
2. 純數字還是允許符號？
3. 建議 PIC：`X(15)`（預留國際格式）

**情境：金額欄位擴充**

1. 現有 PIC：`S9(9)V99`（最大約 10 億）
2. 業務需求：支援更大金額
3. 建議改為：`S9(11)V99` 或 `S9(13)V99`

## 常見錯誤

### 錯誤 1：小數位不足

```cobol
       *> 錯誤：金額只有整數，沒有小數
       01 TRANS-AMT          PIC 9(9).

       *> 正確：金額需要 2 位小數
       01 TRANS-AMT          PIC S9(9)V99.
```

### 錯誤 2：忘記正負號

```cobol
       *> 錯誤：餘額不能為負
       01 ACCT-BALANCE       PIC 9(9)V99.

       *> 正確：餘額可能為負（透支）
       01 ACCT-BALANCE       PIC S9(9)V99.
```

### 錯誤 3：文字欄位當數字用

```cobol
       *> 錯誤：帳號用數字格式，前面的零會消失
       01 ACCT-NO            PIC 9(10).

       *> 正確：帳號應用文字格式，保留前導零
       01 ACCT-NO            PIC X(10).
```

## 重點回顧

| 類型 | PIC 格式 | 說明 |
|------|----------|------|
| 文字 | `X(n)` | n 個任意字元 |
| 數字 | `9(n)` | n 位整數 |
| 帶號數字 | `S9(n)` | 可正負的整數 |
| 小數 | `S9(n)V9(m)` | 帶小數的數字（V 是虛擬小數點） |
| 編輯格式 | `Z`, `$`, `,`, `.` | 輸出格式化用 |
