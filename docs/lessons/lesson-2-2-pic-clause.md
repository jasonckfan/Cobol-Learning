## 一、PIC 子句基礎

### 1.1 什麼是 PIC 子句？

PIC (Picture) 子句是 COBOL 中定義資料格式和長度的關鍵語法。它告訴編譯器這個變數應該如何儲存和顯示。

```
┌─────────────────────────────────────────────────────────────────┐
│              PIC 子句結構                                        │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   層級    變數名稱       PIC      格式定義      其他屬性         │
│   ────    ────────       ───      ────────      ────────         │
│                                                                 │
│   05      CUST-NAME      PIC      X(30).                        │
│   ──      ─────────      ───      ──────                        │
│   層級    變數名稱       關鍵字   30位英數字                     │
│                                                                 │
│   完整範例：                                                     │
│   05  ACCOUNT-BALANCE    PIC  9(9)V99  COMP-3.                  │
│   ──  ───────────────    ───  ────────  ───────                 │
│   層級  變數名稱          PIC   格式定義   儲存格式              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 常用 PIC 符號

```
┌─────────────────────────────────────────────────────────────────┐
│              PIC 符號對照表                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  符號    名稱            用途                    範例           │
│  ────    ────            ────                    ────           │
│                                                                 │
│   X      Alphanumeric    英數字 (字母+數字)       X(10)         │
│   9      Numeric         數字 0-9                9(5)          │
│   A      Alphabetic      字母 A-Z, a-z, 空格     A(20)         │
│   V      Implied Decimal 隱含小數點位置           9(5)V99       │
│   S      Sign            正負號                  S9(7)         │
│   P      Assumed Decimal 隱含小數 (無儲存)        9(3)P(2)      │
│   Z      Zero Suppress   前導零抑制顯示          Z(5)9.99      │
│   ,      Comma           千分位逗號              9,999,999     │
│   .      Period          小數點                  999.99        │
│   $      Dollar Sign     貨幣符號                $$$,$$9.99    │
│   +      Plus Sign       正號                    +999.99       │
│   -      Minus Sign      負號                    -999.99       │
│   CR     Credit          貸方標記 (負數顯示)      999.99CR      │
│   DB     Debit           借方標記 (負數顯示)      999.99DB      │
│   B      Blank           空格插入                9B9B9         │
│   /      Slash           斜線 (日期格式)         99/99/99      │
│   0      Zero            前導零                  00999         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 二、數值資料宣告

### 2.1 整數宣告

```cobol
       *---- 整數宣告範例 -----------------------------------------
       
       * 無正負號整數 (最大 99999)
       05  WS-COUNTER          PIC 9(5).
       
       * 有正負號整數 (範圍 -9999999 到 +9999999)
       05  WS-BALANCE-SIGNED   PIC S9(7).
       
       * 使用 COMP-3 (Packed Decimal) 儲存
       05  WS-AMOUNT-COMP3     PIC 9(9) COMP-3.
       
       * 有正負號 + COMP-3
       05  WS-NET-AMOUNT       PIC S9(11) COMP-3.
```

### 2.2 小數宣告

```cobol
       *---- 小數宣告範例 -----------------------------------------
       
       * 隱含小數點 (儲存為 7位整數+2位小數，共9位)
       * 顯示時自動加小數點
       05  WS-INTEREST-RATE    PIC 9(3)V9(4).
       * 儲存值: 00250000 表示 025.0000 → 25.0000%
       
       * 有正負號的小數
       05  WS-ACCOUNT-BALANCE  PIC S9(13)V99.
       * 範圍: -99,999,999,999,999.99 到 +99,999,999,999,999.99
       
       * 使用 COMP-3 的小數
       05  WS-DAILY-INTEREST    PIC S9(9)V99 COMP-3.
```

### 2.3 數值顯示格式

```cobol
       *---- 數值顯示格式範例 -------------------------------------
       
       * 千分位格式
       05  WS-DISPLAY-AMOUNT   PIC 9,999,999.99.
       * 顯示: 1,234,567.89
       
       * 貨幣格式
       05  WS-CURRENCY-AMOUNT  PIC $$$,$$$,$$9.99.
       * 顯示: $1,234,567.89
       
       * 前導零抑制
       05  WS-SUPPRESS-ZERO    PIC Z(7)9.99.
       * 儲存: 0000123.45 → 顯示: 123.45
       
       * 負數顯示為 CR (Credit)
       05  WS-CREDIT-AMOUNT    PIC 9,999.99CR.
       * 正數: 1,234.56  負數: 1,234.56CR
       
       * 固定正負號
       05  WS-SIGNED-DISPLAY   PIC +9,999.99.
       * 正數: +1,234.56  負數: -1,234.56
```

## 三、文字資料宣告

### 3.1 英數字宣告

```cobol
       *---- 英數字宣告範例 ---------------------------------------
       
       * 純字母 (只能放 A-Z, 空格)
       05  WS-CUSTOMER-NAME    PIC A(30).
       
       * 英數字 (字母+數字+特殊符號)
       05  WS-ADDRESS-LINE     PIC X(50).
       
       * 帳號 (通常英數混合)
       05  WS-ACCOUNT-NUMBER   PIC X(12).
       
       * 產品代碼
       05  WS-PRODUCT-CODE     PIC X(6).
```

### 3.2 日期與時間格式

```cobol
       *---- 日期格式範例 -----------------------------------------
       
       * 標準日期格式 (YYYYMMDD)
       05  WS-TRANSACTION-DATE PIC 9(8).
       * 儲存: 20260411 表示 2026年4月11日
       
       * 顯示格式 (YYYY/MM/DD)
       05  WS-DISPLAY-DATE     PIC 9999/99/99.
       * 顯示: 2026/04/11
       
       * 另一種顯示格式 (DD/MM/YYYY)
       05  WS-DISPLAY-DATE2    PIC 99/99/9999.
       * 顯示: 11/04/2026
       
       * 時間格式 (HHMMSS)
       05  WS-TRANSACTION-TIME PIC 9(6).
       * 儲存: 143025 表示 14:30:25
       
       * 時間顯示 (HH:MM:SS)
       05  WS-DISPLAY-TIME     PIC 99:99:99.
       * 顯示: 14:30:25
```

## 四、FILLER 與常數

### 4.1 FILLER 的使用

FILLER 用於定義不需要單獨命名的欄位，通常用於報表標題、分隔線等。

```cobol
       *---- FILLER 範例 ------------------------------------------
       
       * 報表標題行
       01  WS-REPORT-HEADER.
           05  FILLER            PIC X(20) VALUE 'CUSTOMER REPORT'.
           05  FILLER            PIC X(60) VALUE SPACES.
       
       * 分隔線
       01  WS-DASH-LINE.
           05  FILLER            PIC X(80) VALUE ALL '-'.
       
       * 報表欄位標題
       01  WS-COLUMN-HEADER.
           05  FILLER            PIC X(10) VALUE 'ACCT NUM'.
           05  FILLER            PIC X(2)  VALUE SPACES.
           05  FILLER            PIC X(20) VALUE 'CUSTOMER NAME'.
           05  FILLER            PIC X(2)  VALUE SPACES.
           05  FILLER            PIC X(15) VALUE 'BALANCE'.
           05  FILLER            PIC X(31) VALUE SPACES.
```

### 4.2 VALUE 子句 (初始值)

```cobol
       *---- VALUE 子句範例 ---------------------------------------
       
       * 數值常數
       05  WS-MAX-RECORDS      PIC 9(5)    VALUE 10000.
       05  WS-INTEREST-RATE    PIC 9V9(4)  VALUE 2.5000.
       
       * 文字常數
       05  WS-CURRENCY-CODE    PIC X(3)    VALUE 'USD'.
       05  WS-BANK-NAME        PIC X(30)   VALUE 'ABC BANK'.
       
       * 特殊常數
       05  WS-SPACES           PIC X(100)  VALUE SPACES.
       05  WS-ZEROS            PIC 9(10)   VALUE ZEROS.
       05  WS-HIGH-VALUES      PIC X(100)  VALUE HIGH-VALUES.
       05  WS-LOW-VALUES       PIC X(100)  VALUE LOW-VALUES.
       
       * 條件名稱常數 (Level 88)
       05  WS-ACCOUNT-STATUS   PIC X.
           88  ACCT-ACTIVE               VALUE 'A'.
           88  ACCT-CLOSED               VALUE 'C'.
           88  ACCT-FROZEN               VALUE 'F'.
           88  ACCT-PENDING              VALUE 'P'.
```

## 五、儲存格式 (USAGE)

### 5.1 USAGE 子句說明

USAGE 子句決定資料在記憶體中的儲存格式，影響儲存空間和運算效率。

```
┌─────────────────────────────────────────────────────────────────┐
│              USAGE 格式比較                                      │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  格式          說明                儲存空間        適用場景      │
│  ────          ────                ────────        ────────      │
│                                                                 │
│  DISPLAY       字元顯示格式        1 byte/char     報表輸出      │
│  (預設)        數字每位1 byte                    畫面顯示      │
│                                                                 │
│  COMP          二進位整數          2/4/8 bytes     計數器        │
│  (BINARY)      機器原生格式                      索引值         │
│                                                                 │
│  COMP-1        單精度浮點          4 bytes         科學計算      │
│                                                                 │
│  COMP-2        雙精度浮點          8 bytes         科學計算      │
│                                                                 │
│  COMP-3        壓縮十進位          (n+1)/2 bytes   金額計算      │
│  (PACKED)      每位半 byte                       銀行最常用    │
│                                                                 │
│  COMP-5        原生二進位          依平台而定      系統介面      │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 5.2 金額計算的最佳實踐

```cobol
       *---- 銀行金額計算的最佳實踐 -----------------------------
       
       * ❌ 不建議：使用 DISPLAY 格式計算金額
       05  WS-AMOUNT-DISPLAY   PIC 9(9)V99.
       * 儲存空間: 11 bytes
       * 計算效率: 慢 (需轉換)
       
       * ✅ 建議：使用 COMP-3 計算金額
       05  WS-AMOUNT-COMP3     PIC 9(9)V99 COMP-3.
       * 儲存空間: 6 bytes ( (11+1)/2 )
       * 計算效率: 快 (原生十進位)
       
       * ✅ 建議：顯示時再轉換格式
       05  WS-AMOUNT-WORK      PIC S9(9)V99 COMP-3.
       05  WS-AMOUNT-SHOW      PIC ZZZ,ZZZ,ZZ9.99.
       
       * 計算時使用 WS-AMOUNT-WORK
       * 顯示時 MOVE 到 WS-AMOUNT-SHOW
```

## 六、BA 實務指南

### 6.1 閱讀資料定義的檢查清單

```
┌─────────────────────────────────────────────────────────────────┐
│              BA 資料定義閱讀檢查清單                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  □ 欄位名稱是否清晰？                                            │
│    ├── 是否能從名稱理解用途？                                    │
│    └── 是否符合命名慣例？                                        │
│                                                                 │
│  □ PIC 格式是否正確？                                            │
│    ├── 長度是否足夠？                                            │
│    ├── 小數位數是否正確？                                        │
│    └── 正負號是否考慮？                                          │
│                                                                 │
│  □ 儲存格式是否適當？                                            │
│    ├── 金額欄位是否使用 COMP-3？                                │
│    ├── 計數器是否使用 COMP？                                    │
│    └── 顯示欄位是否使用 DISPLAY？                               │
│                                                                 │
│  □ 是否有條件名稱 (Level 88)？                                  │
│    ├── 條件名稱是否完整？                                        │
│    └── 值定義是否正確？                                          │
│                                                                 │
│  □ 初始值是否合理？                                              │
│    ├── 常數定義是否正確？                                        │
│    └── 預設值是否安全？                                          │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 6.2 常見錯誤與陷阱

| 錯誤類型 | 範例 | 問題 | 修正 |
|----------|------|------|------|
| **長度不足** | `PIC 9(5)` 存帳號 | 帳號可能超過 99999 | `PIC 9(10)` 或 `X(12)` |
| **小數遺失** | `PIC 9(5)` 存利率 | 2.5% 無法表示 | `PIC 9V9(4)` |
| **符號遺失** | `PIC 9(7)` 存餘額 | 無法表示透支 | `PIC S9(7)` |
| **格式錯誤** | `PIC X(9)` 存日期 | 無法進行日期計算 | `PIC 9(8)` |
| **儲存浪費** | `PIC 9(9)V99` DISPLAY | 佔用 11 bytes | 加 `COMP-3` 省空間 |

## 七、練習題

### 練習 1: PIC 格式設計

請為以下欄位設計適當的 PIC 子句：

| 欄位說明 | 需求 | 你的 PIC 設計 |
|----------|------|---------------|
| 客戶姓名 | 中英文姓名，最多30字 | |
| 台幣帳戶餘額 | 最大999萬，需顯示小數點後2位 | |
| 美元匯率 | 如 32.4567，需4位小數 | |
| 交易日期 | YYYYMMDD 格式 | |
| 客戶編號 | 英數混合，固定10碼 | |
| 交易金額 | 可能為負數，最大1億 | |

### 練習 2: 資料宣告分析

分析以下資料宣告，指出潛在問題：

```cobol
01  CUSTOMER-RECORD.
    05  CUST-ID           PIC 9(5).
    05  CUST-NAME         PIC X(20).
    05  CUST-BIRTH-DATE   PIC X(8).
    05  CUST-BALANCE      PIC 9(7)V99.
    05  CUST-STATUS       PIC X.
    05  CUST-EMAIL        PIC X(30).
```

**問題**：
1. CUST-ID 使用 9(5) 有什麼問題？
2. CUST-BIRTH-DATE 使用 X(8) 有什麼缺點？
3. CUST-BALANCE 缺少什麼？
4. CUST-EMAIL 長度是否足夠？

### 練習 3: 計算儲存空間

計算以下變數的儲存空間：

```cobol
01  WS-VARS.
    05  WS-NAME           PIC X(30).
    05  WS-AGE            PIC 9(3).
    05  WS-SALARY         PIC 9(7)V99 COMP-3.
    05  WS-BONUS          PIC S9(5)V99 COMP-3.
    05  WS-DEPARTMENT     PIC X(10).
```

**提示**：
- DISPLAY 格式：1 byte = 1 character
- COMP-3 格式：bytes = (digits + 1) / 2 (無條件進位)

## 八、總結

### 本課程重點回顧

✅ **PIC 符號**: X(英數)、9(數字)、A(字母)、V(隱含小數)、S(正負號)

✅ **數值格式**: 整數、小數、顯示格式、貨幣格式、前導零抑制

✅ **文字格式**: 英數字、日期、時間

✅ **儲存格式**: DISPLAY、COMP、COMP-3 (銀行金額計算首選)

✅ **BA 關注點**: 欄位長度、小數位數、正負號、儲存效率
