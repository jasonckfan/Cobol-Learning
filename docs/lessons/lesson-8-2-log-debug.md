## 一、日誌系統概覽

### 1.1 Mainframe 日誌類型

```
┌─────────────────────────────────────────────────────────────────┐
│              Mainframe 日誌類型                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│   系統層級日誌                                                  │
│   ─────────────                                                 │
│   • SYSLOG: 系統操作日誌                                       │
│   • OPERLOG: 操作員日誌                                        │
│   • LOGREC: 錯誤記錄日誌                                       │
│                                                                 │
│   作業層級日誌                                                  │
│   ─────────────                                                 │
│   • JESMSGLG: JES 訊息日誌                                     │
│   • JESYSMSG: 系統訊息                                         │
│   • SYSPRINT: 程式輸出                                         │
│   • SYSOUT: 一般輸出                                           │
│                                                                 │
│   應用程式日誌                                                  │
│   ─────────────                                                 │
│   • CICS 交易日誌                                              │
│   • DB2 日誌                                                   │
│   • 應用程式自定義日誌                                         │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 SYSLOG

SYSLOG 是系統操作日誌，記錄所有系統活動。

**查看方式**：
- SDSF: LOG 指令
- 系統命令: D LOG

**內容包含**：
- 系統啟動/關閉
- 作業開始/結束
- 錯誤訊息
- 操作員命令

## 二、作業日誌解讀

### 2.1 JESMSGLG (JES Message Log)

包含作業的 JES 相關訊息。

```
範例內容:
─────────
IEF403I CALCBAT - STARTED - TIME=14.30.00
IEF404I CALCBAT - ENDED - TIME=14.35.30
IEF142I CALCBAT STEP010 - STEP WAS EXECUTED
IEF373I STEP /CALCBAT /START 2024123.1430
IEF374I STEP /CALCBAT /STOP  2024123.1435
```

### 2.2 SYSPRINT

程式的主要輸出，通常包含：
- 程式執行訊息
- 報表輸出
- 錯誤訊息
- 統計資訊

### 2.3 常見日誌訊息

| 訊息 | 意義 | 說明 |
|------|------|------|
| **IEF403I** | 作業開始 | STARTED |
| **IEF404I** | 作業結束 | ENDED |
| **IEF142I** | 步驟執行 | STEP WAS EXECUTED |
| **IEF142I RC=0** | 步驟成功 | Return Code 0 |
| **IEF142I RC=4** | 步驟警告 | Return Code 4 |
| **IEF142I RC=8** | 步驟錯誤 | Return Code 8 |
| **IEF142I RC=12** | 嚴重錯誤 | Return Code 12 |
| **IEF450I** | 作業完成 | COMPLETED |

## 三、ABEND 處理

### 3.1 什麼是 ABEND？

ABEND (Abnormal End) 表示程式異常終止。

**常見 ABEND 代碼**：

| 代碼 | 名稱 | 說明 | 常見原因 |
|------|------|------|----------|
| **S0C1** | Operation Exception | 操作異常 | 執行無效指令 |
| **S0C4** | Protection Exception | 保護異常 | 存取無效記憶體 |
| **S0C7** | Data Exception | 資料異常 | 非數字資料計算 |
| **S013** | I/O Error | I/O 錯誤 | 檔案操作錯誤 |
| **S222** | Operator Cancel | 操作員取消 | 作業被強制取消 |
| **S322** | Time Out | 逾時 | 作業超過時間限制 |
| **S806** | Load Failed | 載入失敗 | 程式找不到 |
| **S913** | Security Error | 安全錯誤 | 權限不足 |

### 3.2 ABEND 處理流程

```
發生 ABEND:
───────────
1. 查看日誌確認 ABEND 代碼
2. 檢查 SYSPRINT 錯誤訊息
3. 如有 Dump，分析 Dump
4. 定位錯誤程式位置
5. 修復問題
6. 重新執行測試
```

### 3.3 S0C7 範例分析

**S0C7 - Data Exception** 是最常見的 COBOL ABEND。

**原因**：
- 非數字資料進行數字運算
- COMP-3 資料格式錯誤
- 未初始化的數字欄位

**排查**：
```
1. 查看 Dump 的 PSW (Program Status Word)
2. 找到發生錯誤的指令位址
3. 對應到原始碼行號
4. 檢查相關欄位的值
5. 確認資料格式正確
```

## 四、Dump 分析基礎

### 4.1 什麼是 Dump？

Dump 是程式異常終止時的記憶體傾印，包含：
- 程式狀態
- 變數值
- 呼叫堆疊
- 檔案狀態

### 4.2 Dump 類型

| 類型 | 說明 | 用途 |
|------|------|------|
| **SYSMDUMP** | 系統 Dump | 系統層級分析 |
| **SYSABEND** | ABEND Dump | 程式異常分析 |
| **SYSUDUMP** | User Dump | 使用者定義 Dump |
| **Transaction Dump** | 交易 Dump | CICS 交易分析 |

### 4.3 Dump 分析工具

**IPCSTMVS (Interactive Problem Control System)**：
- 分析 Dump 的互動式工具
- 檢視變數值
- 追蹤呼叫堆疊

**基本指令**：
```
IPCS 常用指令:
──────────────
- LIST: 列出 Dump 內容
- WHERE: 顯示當前位置
- D VAR: 顯示變數值
- D STACK: 顯示呼叫堆疊
- Q SYMPTOM: 查詢錯誤症狀
```

### 4.4 Dump 分析範例

```
症狀: S0C7 Data Exception

分析步驟:
─────────
1. 進入 IPCS
2. 選擇要分析的 Dump
3. 查看 PSW (Program Status Word)
   - 找到發生錯誤的位址
4. 查看呼叫堆疊
   - 確認程式執行路徑
5. 檢查相關變數
   - 確認資料內容
6. 對應原始碼
   - 找到錯誤程式行

常見發現:
─────────
- 變數包含非數字字元 (如 X'40404040' - 空格)
- COMP-3 欄位格式錯誤
- 未初始化的數字欄位
```

## 五、BA 的日誌與除錯角色

### 5.1 BA 在日誌分析中的角色

```
BA 可以處理:
────────────
✓ 查看作業執行狀態
✓ 確認錯誤類型 (ABEND 代碼)
✓ 檢查輸入/輸出筆數
✓ 驗證報表產生
✓ 確認處理時間

BA 需要開發協助:
────────────────
• Dump 深入分析
• 程式邏輯除錯
• 資料格式問題
• 系統層級錯誤
```

### 5.2 問題排查檢查清單

```
作業失敗排查:
──────────────
□ 確認 ABEND 代碼
□ 查看 SYSPRINT 最後訊息
□ 檢查輸入檔案是否存在
□ 確認檔案權限
□ 檢查空間是否足夠
□ 查看是否有資料錯誤
□ 確認程式版本正確
□ 檢查參數設定

資料問題排查:
──────────────
□ 確認輸入資料筆數
□ 檢查輸出資料筆數
□ 比對預期與實際結果
□ 抽樣驗證資料正確性
□ 檢查合計數是否平衡
```

## 六、總結

### 日誌與除錯重點

✅ **日誌類型**: SYSLOG、作業日誌、應用程式日誌

✅ **ABEND 代碼**: S0C7 (資料)、S013 (I/O)、S322 (逾時)

✅ **Dump 分析**: IPCS 工具、變數檢視、堆疊追蹤

✅ **BA 角色**: 初步排查、確認範圍、協助開發
