## 一、USAGE 格式概覽

### 1.1 為什麼需要不同的儲存格式？

```
┌─────────────────────────────────────────────────────────────────┐
│              儲存格式選擇的考量                                  │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  考量因素           DISPLAY      COMP/COMP-5   COMP-3          │
│  ────────           ───────      ───────────   ──────          │
│                                                                 │
│  儲存空間           大           小            最小            │
│  (Space)                                                        │
│                                                                 │
│  計算速度           慢           快            快              │
│  (Speed)           (需轉換)     (機器原生)    (十進位)        │
│                                                                 │
│  可讀性             高           低            低              │
│  (Readability)     (可直接看)   (二進位)      (壓縮)          │
│                                                                 │
│  精度               精確         可能遺失      精確            │
│  (Precision)                                                    │
│                                                                 │
│  銀行適用性         報表輸出     計數器        金額計算        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 1.2 USAGE 格式總表

| 格式 | 名稱 | 儲存方式 | 空間計算 | 最佳用途 |
|------|------|----------|----------|----------|
| **DISPLAY** | 顯示格式 | 字元 (EBCDIC/ASCII) | 1 byte/char | 報表、畫面 |
| **COMP** | 二進位 | 機器整數 | 2/4/8 bytes | 計數器、索引 |
| **COMP-1** | 單精度浮點 | IEEE 754 | 4 bytes | 科學計算 |
| **COMP-2** | 雙精度浮點 | IEEE 754 | 8 bytes | 科學計算 |
| **COMP-3** | 壓縮十進位 | BCD 編碼 | (n+1)/2 bytes | 金額計算 |
| **COMP-5** | 原生二進位 | 機器整數 | 依平台 | 系統介面 |

## 二、DISPLAY 格式

### 2.1 DISPLAY 儲存原理

DISPLAY 是預設格式，每個字元佔用 1 byte，以字元編碼 (EBCDIC 或 ASCII) 儲存。

```
┌─────────────────────────────────────────────────────────────────┐
│              DISPLAY 格式儲存範例                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  宣告: 05  WS-AMOUNT  PIC 9(5)V99.                             │
│                                                                 │
│  數值: 12345.67                                                 │
│                                                                 │
│  儲存內容 (EBCDIC):                                             │
│  ┌────┬────┬────┬────┬────┬────┬────┐                        │
│  │ F1 │ F2 │ F3 │ F4 │ F5 │ F6 │ F7 │                        │
│  └────┴────┴────┴────┴────┴────┴────┘                        │
│    1    2    3    4    5    6    7                             │
│                                                                 │
│  儲存空間: 7 bytes                                              │
│  特點: 可直接閱讀，適合報表輸出                                 │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 DISPLAY 的優缺點

**優點：**
- 人類可讀，除錯方便
- 適合報表輸出和畫面顯示
- 與其他系統交換資料時相容性好

**缺點：**
- 儲存空間浪費 (每個數字 1 byte)
- 計算時需要轉換，效率低
- 不適合大量數值計算

```cobol
       *---- DISPLAY 使用範例 -------------------------------------
       
       * 適合報表輸出
       05  WS-REPORT-AMOUNT    PIC ZZZ,ZZZ,ZZ9.99.
       
       * 適合畫面顯示
       05  WS-SCREEN-DATE      PIC 9999/99/99.
       
       * 適合文字資料
       05  WS-CUSTOMER-NAME    PIC X(50).
       
       * ❌ 不適合大量計算
       05  WS-CALC-AMOUNT      PIC 9(9)V99.     *> 浪費空間且慢
```

## 三、COMP (BINARY) 格式

### 3.1 COMP 儲存原理

COMP 使用二進位整數儲存，依數值大小自動選擇 2、4 或 8 bytes。

```
┌─────────────────────────────────────────────────────────────────┐
│              COMP 格式儲存原理                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  PIC 宣告           儲存空間      數值範圍                      │
│  ─────────          ────────      ────────                      │
│                                                                 │
│  PIC 9(1) TO 9(4)   2 bytes       0 到 9999                     │
│  PIC S9(1) TO S9(4) 2 bytes       -9999 到 +9999                │
│                                                                 │
│  PIC 9(5) TO 9(9)   4 bytes       0 到 999,999,999             │
│  PIC S9(5) TO S9(9) 4 bytes       -999M 到 +999M               │
│                                                                 │
│  PIC 9(10) TO 9(18) 8 bytes       0 到 999... (18位)           │
│  PIC S9(10) TO S9(18) 8 bytes     -999... 到 +999...           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.2 COMP 儲存範例

```
┌─────────────────────────────────────────────────────────────────┐
│              COMP 格式儲存範例                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  宣告: 05  WS-COUNTER  PIC 9(5) COMP.                          │
│                                                                 │
│  數值: 12345                                                    │
│                                                                 │
│  儲存內容 (4 bytes, 二進位):                                    │
│  ┌────┬────┬────┬────┐                                        │
│  │ 00 │ 00 │ 30 │ 39 │                                        │
│  └────┴────┴────┴────┘                                        │
│           (十六進位表示)                                       │
│                                                                 │
│  實際二進位: 00000000 00000000 00110000 00111001               │
│              = 12345 (十進位)                                  │
│                                                                 │
│  儲存空間: 4 bytes (因為 9(5) 超過 4 位數)                     │
│  特點: 計算速度快，但不適合小數                                │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 3.3 COMP 使用場景

```cobol
       *---- COMP 使用範例 ----------------------------------------
       
       * 計數器 (適合)
       05  WS-READ-COUNT       PIC 9(9) COMP.
       05  WS-WRITE-COUNT      PIC 9(9) COMP.
       05  WS-ERROR-COUNT      PIC 9(5) COMP.
       
       * 陣列索引 (適合)
       05  WS-TABLE-INDEX      PIC 9(4) COMP.
       05  WS-LOOP-COUNTER     PIC 9(3) COMP.
       
       * 狀態碼 (適合)
       05  WS-RETURN-CODE      PIC S9(4) COMP.
       
       * ❌ 不適合金額計算 (無法處理小數)
       05  WS-AMOUNT           PIC 9(7) COMP.     *> 錯誤！
```

## 四、COMP-3 (PACKED DECIMAL)

### 4.1 COMP-3 儲存原理

COMP-3 (Packed Decimal) 是銀行業最常用的格式。每個數字用 4 bits (半 byte) 儲存，最後一個 byte 包含最後一個數字和正負號。

```
┌─────────────────────────────────────────────────────────────────┐
│              COMP-3 儲存原理                                     │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  空間計算公式: bytes = (digits + 1) / 2  (無條件進位)          │
│                                                                 │
│  正負號編碼:                                                    │
│  • 正數: 最後半 byte = C (12)                                  │
│  • 負數: 最後半 byte = D (13)                                  │
│  • 無正負號: 最後半 byte = F (15)                              │
│                                                                 │
│  範例: PIC 9(5)V99 COMP-3 儲存 12345.67                        │
│                                                                 │
│  儲存內容:                                                      │
│  ┌───────┬───────┬───────┬───────┐                            │
│  │  1 2  │  3 4  │  5 6  │  7 C  │                            │
│  └───────┴───────┴───────┴───────┘                            │
│   byte 1  byte 2  byte 3  byte 4                               │
│                                                                 │
│  儲存空間: 4 bytes ( (7+1)/2 = 4 )                             │
│  比 DISPLAY 節省: 7 - 4 = 3 bytes (43% 節省)                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.2 COMP-3 儲存範例詳解

```
┌─────────────────────────────────────────────────────────────────┐
│              COMP-3 儲存範例比較                                 │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  範例 1: 正數 12345                                             │
│  ─────────────────────                                          │
│  宣告: PIC 9(5) COMP-3                                          │
│  儲存: 01 23 4F  (3 bytes)                                      │
│  說明: 1-2-3-4-5 + F(無號)                                      │
│                                                                 │
│  範例 2: 正數 +12345                                            │
│  ─────────────────────                                          │
│  宣告: PIC S9(5) COMP-3                                         │
│  儲存: 01 23 4C  (3 bytes)                                      │
│  說明: 1-2-3-4-5 + C(正號)                                      │
│                                                                 │
│  範例 3: 負數 -12345                                            │
│  ─────────────────────                                          │
│  宣告: PIC S9(5) COMP-3                                         │
│  儲存: 01 23 4D  (3 bytes)                                      │
│  說明: 1-2-3-4-5 + D(負號)                                      │
│                                                                 │
│  範例 4: 小數 12345.67                                          │
│  ─────────────────────                                          │
│  宣告: PIC 9(5)V99 COMP-3                                       │
│  儲存: 01 23 45 67 0F  (5 bytes)                                │
│  說明: 1-2-3-4-5-6-7 + F(無號)                                  │
│  注意: V 不佔空間，只是隱含小數點位置                           │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 4.3 為什麼銀行愛用 COMP-3？

```
┌─────────────────────────────────────────────────────────────────┐
│              COMP-3 的銀行優勢                                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 精確計算                                                    │
│     • 十進位儲存，無二進位浮點誤差                              │
│     • 金額計算 100% 準確                                        │
│     • 符合會計要求                                              │
│                                                                 │
│  2. 節省空間                                                    │
│     • 比 DISPLAY 節省 40-50% 空間                               │
│     • 大量資料時效益明顯                                        │
│                                                                 │
│  3. 計算效率                                                    │
│     • 硬體直接支援十進位運算                                    │
│     • 無需格式轉換                                              │
│     • 適合大量批次計算                                          │
│                                                                 │
│  4. 監管合規                                                    │
│     • 符合金融業計算精度要求                                    │
│     • 稽核追蹤容易                                              │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

## 五、格式比較與選擇

### 5.1 儲存空間比較

| 宣告 | DISPLAY | COMP | COMP-3 | 最佳選擇 |
|------|---------|------|--------|----------|
| `9(5)` | 5 bytes | 4 bytes | 3 bytes | COMP-3 |
| `9(9)` | 9 bytes | 4 bytes | 5 bytes | COMP |
| `S9(7)V99` | 9 bytes | - | 5 bytes | COMP-3 |
| `9(3)` | 3 bytes | 2 bytes | 2 bytes | COMP |
| `X(20)` | 20 bytes | - | - | DISPLAY |

### 5.2 銀行常見欄位建議

```cobol
       *---- 銀行欄位最佳實踐 -------------------------------------
       
       * 金額欄位 - 必須使用 COMP-3
       05  ACCT-BALANCE        PIC S9(13)V99 COMP-3.
       05  TXN-AMOUNT          PIC S9(11)V99 COMP-3.
       05  INT-ACCrued         PIC S9(9)V99 COMP-3.
       
       * 利率欄位 - 使用 COMP-3
       05  INTEREST-RATE       PIC 9V9(4) COMP-3.

       05  RECORD-COUNT        PIC 9(9) COMP.
```

## 六、計算精度問題

### 6.1 浮點數的危險

```cobol
       *---- 浮點數精度問題示範 -----------------------------------
       
       * ❌ 錯誤：使用 COMP-1 (單精度浮點)
       05  WS-AMOUNT-FLOAT     COMP-1.
       
       * 計算結果可能不精確：
       * 0.1 + 0.2 可能 = 0.30000001192092896
       
       * ✅ 正確：使用 COMP-3
       05  WS-AMOUNT-PACKED    PIC S9(9)V99 COMP-3.
       
       * 計算結果精確：
       * 0.1 + 0.2 = 0.30 (完全精確)
```

### 6.2 中間結果精度

```cobol
       *---- 中間結果精度處理 -------------------------------------
       
       * 利息計算範例
       05  WS-PRINCIPAL        PIC S9(11)V99 COMP-3.
       05  WS-RATE             PIC 9V9(4) COMP-3.
       05  WS-DAYS             PIC 9(3) COMP.
       05  WS-INTEREST         PIC S9(9)V99 COMP-3.
       
       * 計算公式
       COMPUTE WS-INTEREST = 
           WS-PRINCIPAL * WS-RATE * WS-DAYS / 365.
       
       * ⚠️ 注意：中間結果可能溢出！
       * WS-PRINCIPAL (13位) * WS-RATE (5位) = 18位
       * 可能超過 WS-INTEREST 的 11 位限制
       
       * ✅ 安全的做法：使用更大的中間變數
       05  WS-TEMP-CALC        PIC S9(15)V99 COMP-3.
       
       COMPUTE WS-TEMP-CALC = 
           WS-PRINCIPAL * WS-RATE * WS-DAYS / 365.
       MOVE WS-TEMP-CALC TO WS-INTEREST.
```

## 七、BA 實務指南

### 7.1 資料格式審查檢查清單

```
┌─────────────────────────────────────────────────────────────────┐
│              資料格式審查檢查清單                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  □ 金額欄位                                                     │
│    ├── 是否使用 COMP-3？                                        │
│    ├── 整數位數是否足夠？ (通常 9-13 位)                        │
│    ├── 小數位數是否正確？ (通常 2 位)                           │
│    └── 是否有正負號 (S)？                                       │
│                                                                 │
│  □ 計數器欄位                                                   │
│    ├── 是否使用 COMP？                                          │
│    ├── 範圍是否足夠？                                           │
│    └── 是否需要正負號？                                         │
│                                                                 │
│  □ 文字欄位                                                     │
│    ├── 長度是否足夠？                                           │
│    ├── 是否考慮多語言？                                         │
│    └── 是否使用 DISPLAY (預設即可)？                            │
│                                                                 │
│  □ 計算精度                                                     │
│    ├── 中間結果是否可能溢出？                                   │
│    ├── 除法運算的餘數處理？                                     │
│    └── 四捨五入規則是否明確？                                   │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 7.2 常見錯誤與修正

| 錯誤 | 問題 | 修正 |
|------|------|------|
| `9(9)V99` DISPLAY | 浪費空間，計算慢 | 改為 `S9(9)V99 COMP-3` |
| `S9(7) COMP` 存金額 | 無法處理小數 | 改為 `S9(7)V99 COMP-3` |
| `9(5)V99 COMP-3` 無 S | 無法表示負數 | 加 S: `S9(5)V99 COMP-3` |
| `COMP-1` 存金額 | 精度遺失 | 改為 `COMP-3` |
| `9(3)` 存大額計數 | 可能溢出 | 改為 `9(9) COMP` |

## 九、總結

### 本課程重點回顧

✅ **DISPLAY**: 字元儲存，適合報表和文字，1 byte/char

✅ **COMP**: 二進位整數，適合計數器，2/4/8 bytes

✅ **COMP-3**: 壓縮十進位，銀行金額計算首選，(n+1)/2 bytes

✅ **銀行最佳實踐**: 金額用 COMP-3，計數器用 COMP，文字用 DISPLAY

✅ **精度重點**: 避免浮點數，注意中間結果溢出
