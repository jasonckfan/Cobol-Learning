# BA 溝通指南

> 與開發人員討論需求時的實用提問技巧

---

## 需求分析時必問的問題

### 1. 輸入來源

| 要確認的事 | 提問方式 |
|-----------|---------|
| 資料從哪來 | 「這個欄位的來源是什麼？從哪個系統？哪個檔案？」 |
| 檔案格式 | 「來源檔案是 Sequential 還是 VSAM？固定長度還是變動長度？」 |
| 欄位定義 | 「欄位規格在哪個 Copybook？可以給我看一下嗎？」 |
| 資料量 | 「一天大概多少筆資料？尖峰時段呢？」 |

### 2. 處理邏輯

| 要確認的事 | 提問方式 |
|-----------|---------|
| 處理方式 | 「這個邏輯是 Online 即時處理，還是放 Batch？」 |
| 條件判斷 | 「有哪些條件分支？每個分支的處理方式是什麼？」 |
| 例外處理 | 「如果資料異常，程式怎麼處理？會寫錯誤檔嗎？」 |
| 計算公式 | 「計息公式是怎麼算的？有無四捨五入邏輯？」 |

### 3. 輸出結果

| 要確認的事 | 提問方式 |
|-----------|---------|
| 輸出到哪 | 「結果寫入哪個檔案？更新哪個 Table？」 |
| 報表格式 | 「報表的欄位順序、格式是什麼？」 |
| 通知機制 | 「處理完成後要通知誰？有無 alert 機制？」 |

## 影響分析時必問的問題

### 修改現有功能時

```
「如果改這個欄位長度，想確認幾件事：」

1. 「有哪些程式用到這個欄位？」
2. 「Copybook 要改嗎？會不會影響其他程式？」
3. 「檔案格式要改嗎？上游/下游系統知道嗎？」
4. 「歷史資料要不要轉檔？」
5. 「現有的報表會不會跑版？」
```

### 新增功能時

```
「這個新功能想確認幾件事：」

1. 「要新增程式還是修改現有程式？」
2. 「要新增檔案還是用現有檔案？」
3. 「要排在哪個 Job？哪個時間點執行？」
4. 「上下游 Job 有沒有依賴關係？」
5. 「測試環境的資料準備好了嗎？」
```

## 測試設計時必問的問題

### 測試案例設計

| 情境 | 提問方式 |
|-----|---------|
| 條件分支 | 「程式裡有幾個 IF/ELSE？每個分支都要測到嗎？」 |
| 邊界值 | 「金額上限是多少？超過會怎樣？」 |
| 例外情境 | 「如果檔案是空的，程式會怎樣？」 |
| 批次測試 | 「這個 Job 要怎麼測？可以手動觸發嗎？」 |

### 缺陷分析時

```
「這個 defect 想確認一下：」

1. 「是程式邏輯問題，還是資料問題？」
2. 「是這支程式的問題，還是上游傳來的資料就錯了？」
3. 「修正後，要不要回補歷史資料？」
4. 「有沒有其他類似的程式也有這個問題？」
```

## 常見誤解澄清

### 當開發說「這個改不了」

可能的真實原因：
- 技術上可以改，但風險很高
- 牽動太多程式，工時成本高
- 沒有測試環境可以驗證
- 擔心影響其他系統

**你應該問：**
- 「是技術限制還是風險考量？」
- 「如果硬要改，風險在哪裡？」
- 「有沒有替代方案可以達到類似效果？」

### 當開發說「這要放批次」

可能的含義：
- 無法即時處理（效能問題）
- 需要等待其他資料
- 牽涉日終結算邏輯
- 開發工時較短

**你應該問：**
- 「為什麼不能 Online 即時處理？」
- 「如果使用者期望即時看到，有沒有折衷方案？」
- 「批次排在什麼時間？使用者能接受嗎？」

## 溝通範例

### 範例 1：新增計息規則

**業務需求：** 新增一種「階梯式利率」，依帳戶餘額分級計息。

**BA 提問清單：**

```
【輸入】
1. 利率參數從哪裡讀取？目前有這個參數檔嗎？
2. 帳戶餘額是用當日餘額還是平均餘額？
3. 這個規則適用哪些帳戶類型？要怎麼判斷？

【處理】
4. 要修改哪支計息程式？
5. 階梯的判斷邏輯是怎樣？有幾級？
6. 如果當月有異動，要怎麼處理？

【輸出】
7. 利息明細報表要加欄位嗎？
8. 對帳邏輯要不要改？

【測試】
9. 測試資料怎麼準備？需要開測試帳戶嗎？
10. 怎麼驗證計息正確？有沒有試算工具？
```

### 範例 2：報表欄位調整

**業務需求：** 資金池報表新增「歸集金額」欄位。

**BA 提問清單：**

```
【輸入】
1. 歸集金額的來源是哪個檔案？哪個欄位？
2. 這個欄位目前有抓嗎？還是要新加？

【處理】
3. 報表程式要改嗎？還是 Copybook 要改？
4. 如果 Copybook 改了，會影響其他程式嗎？

【輸出】
5. 新欄位要放在報表的哪個位置？
6. 報表格式改了，下游有沒有系統在讀這個報表？

【測試】
7. 有沒有測試報表可以對照？
8. 格式改了，報表總長度會不會超過限制？
```

## 常用文件清單

跟開發要文件時，可以用這些名稱：

| 文件類型 | 用途 | 問法 |
|---------|------|------|
| 欄位規格書 | 確認欄位定義 | 「可以給我這個檔案的欄位規格書嗎？」 |
| Copybook | 看欄位宣告 | 「這個檔案的 Copybook 在哪裡？」 |
| 程式規格書 | 看處理邏輯 | 「這支程式的規格書可以給我看嗎？」 |
| JCL | 看批次流程 | 「這個 Job 的 JCL 可以給我嗎？」 |
| Job Flow | 看批次順序 | 「日終批次的 Job Flow 可以給我嗎？」 |
| 介面規格書 | 看系統介接 | 「上下游系統的介面規格書在哪裡？」 |
